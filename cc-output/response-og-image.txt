2. app/page.tsx  [MODIFIED]
   Removed: openGraph.images: [{ url: '/og-image.png' }]
   Removed: twitter.images: ['/og-image.png']
   Reason:  app/opengraph-image.tsx (file-based convention) automatically injects
            og:image and twitter:image meta tags. Explicit images[] in the metadata
            config would prevent the file-based image from being used. Removing them
            lets opengraph-image.tsx be the single source of truth.

   The /og-image.png reference in public/ is now unused.
   Safe to delete public/og-image.svg and leave public/og-image.png as fallback,
   or delete both since the dynamic route handles generation.

How it renders at runtime:
  GET /opengraph-image
    → Next.js invokes OgImage() server-side
    → Returns PNG via ImageResponse (Satori under the hood)
  Meta tags auto-injected:
    <meta property="og:image" content="/opengraph-image" />
    <meta name="twitter:image" content="/opengraph-image" />

=== FIX 1 — Add crons to vercel.json ===

Add this block to vercel.json (inside the root JSON object):

  "crons": [
    {
      "path": "/api/cron/daily-deals",
      "schedule": "0 7 * * *"
    },
    {
      "path": "/api/cron/weekly-digest",
      "schedule": "0 8 * * 1"
    }
  ]

Schedules use UTC. Adjust for IST (UTC+5:30):
  daily-deals  → 0 7 * * *    runs daily at 12:30 IST
  weekly-digest → 0 8 * * 1   runs Monday at 13:30 IST

IMPORTANT: Vercel cron jobs are authenticated via the Vercel platform —
  the Authorization: Bearer {CRON_SECRET} header is NOT sent by Vercel natively.
  You must either:
    a) Use the Vercel Cron Secret (CRON_SECRET env var set in Vercel dashboard)
       and verify it in the route — Vercel injects it as Authorization: Bearer
    b) OR remove the x-cron-secret check and rely on Vercel's built-in auth

  Vercel sends: Authorization: Bearer <CRON_SECRET from dashboard>
  Current code reads: x-cron-secret header
  These do NOT match — fix the header check too.

=== FIX 2 — Harden secret check (apply to both routes) ===

BEFORE:
  const secret = r.headers.get('x-cron-secret')
  if (env.CRON_SECRET && secret !== env.CRON_SECRET) return ...

AFTER (aligns with Vercel's Authorization header pattern):
  const authHeader = r.headers.get('authorization')
  const secret = authHeader?.startsWith('Bearer ') ? authHeader.slice(7) : null
  if (!secret || secret !== env.CRON_SECRET) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

=== FIX 3 — Add error handling (apply to both routes) ===

    return NextResponse.json({ ok: true, dealsCount: deals.length, triggeredAt: new Date().toISOString() })
  } catch (err) {
